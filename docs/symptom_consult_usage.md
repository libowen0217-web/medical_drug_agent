# 症状问诊辅助模块使用说明

## 为什么新增这个模块

这条链路用于做症状相关的辅助初筛，不是诊断系统，也不会自动开药。  
它先筛查红旗症状，再在没有明显红旗信号时给出可供医生或药师参考的 OTC 候选类别，并复用现有药物安全多 Agent 链路做候选药安全检查。

## 它不做什么

- 不做疾病诊断
- 不自动开药
- 不直接说“应该吃什么药”
- 不直接说“可以吃/不能吃”
- 不推荐抗生素、处方药、激素等需要医生确认的药物

## 红旗症状的作用

如果命中红旗症状，例如：

- 呼吸困难
- 胸痛
- 意识模糊
- 咳血
- 高热不退

系统会停止 OTC 候选展示，并提示由医生进一步评估，必要时及时就医。

## OTC 候选药的安全边界

当前只提供医生/药师参考的 OTC 候选类别：

- 退热镇痛类
- 抗过敏类
- 化痰/祛痰类

每个候选药物仍会进入现有药物安全检查链路，分析与当前用药、基础疾病、年龄和风险规则之间的关系。

## 与原药物安全链路的关系

- 症状问诊辅助链路：做红旗筛查与 OTC 候选初筛
- 原药物安全链路：做候选药与当前用药之间的安全检查

两条链路职责不同，但症状模块会复用原多 Agent 药物安全链路做候选药安全分析。

## 运行方式

```bash
python scripts/run_symptom_case.py
python scripts/run_symptom_case.py --input scripts/sample_symptom_case.json
python scripts/run_symptom_case.py --red-flag-demo
```

## FastAPI 调用

```http
POST /api/v1/symptom-consult/check
```

请求示例：

```json
{
  "symptom_text": "发热、头痛、嗓子疼，两天了",
  "age": 68,
  "sex": "unknown",
  "temperature_c": 38.5,
  "duration_days": 2,
  "current_drugs": ["硝苯地平"],
  "diseases": ["高血压"],
  "patient_factors": [],
  "allergies": []
}
```

## 当前限制

- 仅支持常见感冒/发热/咳嗽/鼻部症状的规则化处理
- 仅支持本地规则
- 不接真实医生问诊系统
- 不接外网医学 API
- 不给处方建议

## 后续升级方向

- 增加更多常见症状规则
- 增加真实说明书或证据摘要
