# Streamlit 前端使用说明

## 总体定位

当前 Streamlit 页面已经重构为更接近现代医疗系统的卡片式界面，主路径仍然围绕多智能体能力展示。

页面标题：

`社区药店多智能体用药安全辅助系统`

副标题：

`基于多智能体协作、药物安全规则、剂量检查、本地 RAG 证据和候选药协作评估的用药安全辅助系统`

## 为什么不在主界面并列展示三种后端模式

后端仍保留：

- 默认 service 路径
- LangGraph 路径
- 多智能体路径

但前端主界面默认聚焦多智能体主路径，避免项目目标被“多种实现方式”稀释。这样更符合当前系统“多智能体社区药店用药安全辅助”的展示重点。

## 启动方式

```bash
python scripts/run_streamlit.py
```

或：

```bash
streamlit run medical_drug_agent/ui/streamlit_app.py
```

启动成功后通常可访问：

- `http://localhost:8501`
- `http://127.0.0.1:8501`

## 4 个 Tab 的作用

1. `药物交互检查`
   默认调用 `SupervisorAgent`，从药库中选择当前用药与拟新增药物，展示风险发现、证据引用、药师版报告和患者版报告。

2. `症状问诊辅助`
   默认调用 `SymptomConsultWorkflow`，展示症状解析、红旗症状筛查、OTC 候选药、候选药安全结果和问诊辅助报告。

3. `候选药协作辩论`
   优先读取最近一次症状问诊结果中的 `medication_debate_summary` 与 `debate_results`，集中展示多 Agent 协作评估。

4. `系统调试与审计`
   展示最近一次药物交互检查和症状问诊的 JSON、审计信息和智能体执行链路。

## 页面优化点

- 顶部“设置”弹层替代原来的大面积侧边栏
- 所有按钮与输入组件使用唯一 key
- 药物输入支持从当前药库搜索选择
- 特殊患者因素增加合理性校验
- 页面主要用户可见内容尽量中文化

## 演示方式

可以直接演示：

- 药物交互检查
- 症状问诊辅助
- 红旗症状阻断
- 候选药协作辩论
- evidence 证据引用
- LLMReportAgent 可选润色
- 审计信息与 JSON 调试

## 医疗安全边界

- 不做疾病诊断
- 不开药
- 不替代医生或药师判断
- 不把 `preferred_candidate` 解释成推荐用药
- 不输出“应该吃 / 可以吃 / 放心服用 / 完全安全”等不安全表述
- 红旗症状触发时，会明确阻断 OTC 候选药展示和候选药协作评估
